# nifty-filter

nifty-filter is a read-only NixOS router distribution with an
nftables firewall, DHCP, and DNS. Configuration is driven entirely by
env files on the writable `/var` partition. The root filesystem is
read-only.

The nftables ruleset is generated by a Rust binary using a
compile-time validated [template](templates/router.nft.txt) powered by
[askama](https://github.com/djc/askama).

## Quick start

### 1. Build the ISO

```bash
nix build .#iso
```

Or use the Justfile:

```bash
just iso
```

### 2. Boot the ISO

Flash to USB or attach to a VM:

```bash
sudo dd if=result/iso/nifty-filter-*.iso of=/dev/sdX bs=4M status=progress
```

Boot from the media. The console shows the IP address and login
credentials (`admin` / `nifty`).

### 3. Install

From your workstation, copy your SSH key and connect:

```bash
ssh-copy-id admin@<router-ip>
ssh admin@<router-ip>
```

Run the interactive installer:

```bash
nifty-install
```

The installer prompts for:

 * Hostname
 * Target disk
 * WAN interface (upstream/internet)
 * LAN interface (local network)
 * LAN subnet and DHCP pool
 * DNS servers

Interfaces are renamed to `wan` and `lan` by MAC address so the
configuration is portable across hardware. The system powers off after
install — remove the media and power on.

### 4. Configure

SSH into the installed system:

```bash
ssh admin@<router-ip>
```

Use the interactive configuration menu:

```bash
nifty-config
```

Or edit the env files directly:

```bash
nano /var/nifty-filter/router.env
nano /var/nifty-filter/dhcp.env
```

Apply firewall changes without rebooting:

```bash
sudo systemctl restart nifty-filter
```

Apply DHCP/DNS changes:

```bash
sudo systemctl restart nifty-dnsmasq
```

## System architecture

### Filesystem layout

| Mount | Mode | Purpose |
|-------|------|---------|
| `/` | read-only | NixOS system, nifty-filter binary, all services |
| `/var` | read-write | Router config, DHCP leases, SSH keys, logs |
| `/boot` | read-write | EFI system partition, kernel, bootloader |
| `/tmp` | tmpfs | Scratch (cleared on reboot) |
| `/home` | bind from `/var/home` | User home directories |

### Boot services

| Service | Purpose | Config source |
|---------|---------|---------------|
| `nifty-link` | Renames interfaces by MAC address | `/var/nifty-filter/network/*.link` |
| `nifty-hostname` | Sets hostname | `router.env` |
| `nifty-network` | Configures WAN (DHCP) and LAN (static IP) | `router.env` |
| `nifty-filter-init` | Seeds default config on first boot | — |
| `nifty-filter` | Generates and applies nftables rules | `router.env` |
| `nifty-dnsmasq` | DHCP and DNS server | `dhcp.env` |

### Configuration files

All config lives in `/var/nifty-filter/`:

```
/var/nifty-filter/
  router.env              # Firewall, interfaces, hostname
  dhcp.env                # DHCP pool, DNS servers
  network/
    10-wan.link           # WAN interface rename rule
    10-lan.link           # LAN interface rename rule
  ssh/
    ssh_host_*            # Persistent SSH host keys
```

### Default firewall rules

 * Default deny on input, forward, and output chains
 * Stateful connection tracking (established/related)
 * LAN-to-WAN forwarding with masquerade NAT
 * SSH (port 22) accepted on both WAN and LAN
 * DHCP (ports 67/68) accepted on LAN
 * ICMP echo accepted on LAN
 * Configurable port forwarding (DNAT)

## Router commands

| Command | Description |
|---------|-------------|
| `nifty-config` | Interactive configuration menu |
| `nifty-upgrade` | Pull latest source, build, and reboot |
| `nifty-maintenance` | Reboot into maintenance mode (rw root) |

## Upgrading

### From the router

```bash
nifty-upgrade
```

This temporarily remounts the filesystem read-write, pulls the latest
source from git, builds the new system closure, updates the boot
entries, and reboots.

### From a workstation

```bash
just upgrade <router-ip>
```

Builds locally, rsyncs changed store paths to the router over SSH,
updates boot entries, and reboots.

## Maintenance mode

For manual system changes, boot into maintenance mode:

```bash
nifty-maintenance
```

This sets a one-shot boot entry with the root filesystem mounted
read-write. The console auto-logs in and shows a red `[MAINTENANCE]`
prompt. After maintenance, reboot to return to normal read-only mode.

## Included services

 * **nifty-filter** — nftables firewall with IP forwarding and NAT
 * **dnsmasq** — DHCP server and DNS forwarder (Cloudflare by default)
 * **OpenSSH** — key-only auth, no root login, persistent host keys

## Standalone usage

nifty-filter can also be used as a standalone binary on any Linux
system to generate nftables rules.

### Install

```bash
cargo install nifty-filter
```

Or [download a release](https://github.com/EnigmaCurry/nifty-filter/releases).

### Usage

```bash
# Generate rules from an env file:
nifty-filter nftables --env-file router.env --strict-env

# Generate and validate:
nifty-filter nftables --env-file router.env --strict-env --validate

# From environment variables:
INTERFACE_LAN=lan INTERFACE_WAN=wan SUBNET_LAN=10.99.0.1/24 \
  nifty-filter nftables
```

See [examples/](examples/) for complete configurations.
